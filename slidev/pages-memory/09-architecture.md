# Шпаргалка: що куди

```
ЗАДАЧА                                   РІШЕННЯ
─────────────────────────────────────────────────────────────────────
Пам'ятати розмову (≤ 20 повідомлень)  → ConversationBufferWindowMemory
Пам'ятати розмову (30+ повідомлень)   → ConversationSummaryBufferMemory
Persistent між сесіями                → LangGraph + SqliteSaver
Знання домену / документи             → RAG: Chroma / Qdrant + Embeddings
Точний пошук + семантичний            → Hybrid: BM25 + Embedding + RRF
Правила / SOP / обмеження             → CLAUDE.md або Markdown файли
Few-shot приклади                      → RAG по минулих задачах
Персоналізація конкретного юзера      → Mem0 self-hosted / Zep
Самонавчання (безпечно)               → HITL: prepare → confirm → save
Автономний агент (тривалий)           → MemGPT/Letta
Кешування дорогих обчислень           → Redis / better-sqlite3 + TTL
Кешування статичного промпту          → KV Cache (cache_control)
Skills замість 50 MCP tools           → Одна точка входу → маршрутизація
```

**Антипатерни яких уникати:**

| ❌ Антипатерн | ✅ Правильно |
|---------------|-------------|
| Все в системний промпт | RAG: знаходь тільки потрібне |
| Одна колекція для всього | Окремі колекції за типом даних |
| Кеш без TTL | Redis/pickle + TTL (час придатності) |
| Auto-save без підтвердження | HITL: prepare → confirm → save |
| Замінити embedding модель "на льоту" | Rebuild індексу з нуля |
