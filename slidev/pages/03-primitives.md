# 🔧 Tools — виконувані функції

Модель **викликає** tool і отримує результат

```js
// Визначення tool на сервері
server.tool(
  "search_github",                              // назва
  { query: z.string(), repo: z.string() },      // параметри (з типами)
  async ({ query, repo }) => {                  // виконання
    const results = await githubApi.search(query, repo);
    return { content: [{ type: "text", text: JSON.stringify(results) }] };
  }
);
```

<v-click>

Модель бачить тільки **назву** + **опис** + **схему параметрів**.  
За описом вирішує — чи потрібен цей tool для відповіді на запит.

</v-click>

<!--

# 📂 Resources — джерела даних

Модель **читає** ресурс — статичні або динамічні дані

```
file://project/README.md          → вміст файлу
database://production/users/schema → схема таблиці
api://weather/kyiv                 → поточна погода
```

<v-click>

**Різниця від Tools:**
- `Tool` → модель **викликає**, отримує результат дії
- `Resource` → модель **читає**, отримує дані (контекст)

</v-click>

<!--
Приклад: Tool = "відправити email", Resource = "прочитати inbox".
Resources підходять для надання контексту моделі на початку розмови.
-->

---

# 📝 Prompts — шаблони взаємодії

Готові структуровані шаблони для типових задач

```
code-review    → підготовлений prompt для ревʼю коду
summarize      → шаблон для підсумовування документа
generate-tests → prompt для генерації тестів
```

<v-click>

**Сьогодні використовуємо тільки Tools.**  
Resources і Prompts — для більш складних кейсів.

</v-click>
